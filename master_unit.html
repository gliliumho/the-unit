Requirements

The master unit can be replaced with any embedded boards or even PC. All the codes so far are written using Python 3 so it can run on any machine as long as it has the Python 3 interpreter.

Software Requirements

The following software are needed:

  • Python 3 (version 3.5 or newer)
  • PuTTY for Windows / minicom for Linux

Hardware Requirements

On the hardware side, you'll need the following:

  • UART ports or..
  • at least 1x USB port for 3-pin UART/TTL to USB cable
  • Ethernet or Wi-Fi for networking

What does it do?

The master unit can be considered a 'gateway' between the slave units and other devices (such as PC, server, Internet and etc). The master unit will be connected to masterRF (master for RF board) through UART. The master does the following:

  • Receive traffic info from traffic servers
  • Send traffic info to slave units
  • Request heartbeat from slave units and log their status
  • (not done) TCP server to send information about slave units (such as heartbeat and sensor data)

Architecture

Since master unit is the gateway between slave units and other devices, you may refer to other sensor networks for similar architecture to better understand it.

Only one master unit is needed for each side of the highway. So if you've got a project (hahahahhaah) to deploy The Unit at both sides of the highway of 300km, it'll need two master units....and probably thousands of slave units.
